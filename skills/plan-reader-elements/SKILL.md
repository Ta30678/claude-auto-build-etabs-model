# 結構構件辨識與命名規則

本技能涵蓋結構配置圖中構件的辨識、命名、圖例解讀、Grid 座標系統、特殊柱位處理。

---

## 一、構件命名規則

### 1.1 柱 (Column) — 前綴 "C"

```
格式：C{X向寬}X{Y向深}
範例：C90X90, C100X110, C80X60
```

- X向寬 = Global X 方向（水平）長度 (cm)
- Y向深 = Global Y 方向（垂直）長度 (cm)
- 圖面上以填色方塊或矩形表示，位於 Grid Line 交叉點

### 1.2 大梁 (Main Beam) — 前綴 "B"

```
格式：B{寬}X{深}
範例：B55X80, B70X80, B60X90
```

- 寬 = 梁寬 (cm)，深 = 梁深/梁高 (cm)
- 圖面上以粗色線條表示，沿 Grid Line 連接柱與柱

### 1.3 小梁 (Secondary Beam) — 前綴 "SB"

```
格式：SB{寬}X{深}  或  SB{寬1}/{寬2}X{深}
範例：SB35X65, SB30X60, SB35/40X65
```

- SB35/40X65 表示同系列有兩種寬度 (35cm, 40cm)，深度相同
- 圖面上以色線條表示，可能比大梁細
- 位於大梁之間，不一定在 Grid Line 上
- **位置必須從 annotation.json 或圖面像素量測取得**

### 1.4 壁梁 (Wall Beam) — 前綴 "WB"

```
格式：WB{寬}X{深}
範例：WB50X70, WB40X60
```

- 位於剪力牆頂部，有時顏色較深（棕色、深紅等）

### 1.5 剪力牆 (Shear Wall)

```
格式：{厚度}cm 剪力牆  或  SW{厚度}
範例：50cm 剪力牆, SW25
```

- 圖面上以粗矩形色塊/色帶表示，沿 Grid Line 配置

### 1.6 連續壁 (Diaphragm Wall)

```
格式：{厚度}cm 連續壁  或  DW{厚度}
範例：80cm 連續壁, DW80
```

- **材質一律使用 C280**（除非使用者另外指定）
- 通常出現在地下室外圍

### 1.7 基礎梁 (Foundation Beam) — 前綴 "FB"

```
格式：FB{寬}X{深}
範例：FB90X230, FB80X200
```

- 僅出現在基礎層，尺寸通常較大

### 1.8 基礎小梁 / 基礎邊梁 — 前綴 "FSB" / "FWB"

```
FSB{寬}X{深}  範例：FSB40X80
FWB{寬}X{深}  範例：FWB40X80
```

## 二、圖例 (Legend) 解讀方法

### 2.1 解讀流程

```
Step 1: 找到圖例區域（通常在圖面左側、右側或下方空白處）
Step 2: 逐項讀取（圖形類型 + 文字標註）
        方塊 = 柱、線條 = 梁、粗帶 = 牆
Step 3: 建立「圖面顏色/圖示 → 構件」對照表
        此對照表僅適用於當前這張圖
Step 4: 用此對照表辨識圖面上所有構件
```

### 2.2 判讀要點

- **不要假設顏色含義** — 紅色不一定是柱，藍色不一定是大梁，以圖例為準
- 圖例中的文字 = 構件尺寸名稱，直接讀取
- 不同案子、不同樓層可能使用不同色彩

### 2.3 無圖例時的處理

1. 從圖面文字標註直接讀取構件名稱
2. 根據構件形狀推斷類型（方塊→柱、線條→梁）
3. **主動詢問用戶**確認對照關係

## 三、Grid 座標系統

### 3.1 基本規則

```
X方向 (水平)：通常用數字編號 → 1, 2, 3, 4, 5 ...
Y方向 (垂直)：通常用英文字母編號 → A, B, C, D ...
```

- Grid Line 間距標示在圖面邊緣（單位 cm 或 mm）
- 圈號 (Grid Bubble) 為圓圈內的數字或字母

### 3.2 構件位置描述

```
柱位：   Grid 交叉點 → "Grid C-3 柱" 或 "(C,3)"
梁跨度： 起終點 Grid → "Grid A~D / Grid 2 大梁"
牆位：   Grid 區段   → "Grid B~C / Grid 1 剪力牆"
```

### 3.3 Grid 間距與累積座標

從圖面邊緣尺寸標註讀取，用於小梁定位：

```
範例：
X方向 Grid: 1=0, 2=860, 3=1720, 4=2580 (cm)
Y方向 Grid: A=0, B=700, C=1400, D=2100 (cm)
```

**這些間距和座標是計算小梁位置的基礎，必須精確記錄。**

## 四、特殊柱位處理

### 4.1 退縮柱位

柱子從某樓層向內退縮時：
- **必須詢問使用者**處理方式（轉換柱/取消柱/其他）
- 未獲指示前不自行決定

### 4.2 斜柱 / 斜撐

- ETABS 中使用 **BRACE** 建立，不是 COLUMN
- 注意柱連續性不能斷開
- 起終點座標從圖面精確讀取
- 輸出中標註為 `BRACE`

### 4.3 柱連續性檢查

```
每根柱從基礎到屋頂應保持連續：
  B3F → B2F → B1F → 1F → 2F → ... → RF

如果某樓層柱位消失或偏移：
  1. 標記為「不連續」
  2. 詢問使用者處理方式
  3. 確認是否需要轉換梁/斜撐
```

## 五、共構下構範圍判讀

### 5.1 背景

多棟共構（如 3 棟共用地下室）通常只建一棟模型。下構範圍比上構大，需判讀邊界處理方式。

### 5.2 三種情況

| 情況 | 條件 | 處理 |
|------|------|------|
| 1 | 上構柱區已是最外邊 | 直接延續往下建 |
| 2 | 下構比上構大，最外邊不在 Grid Line 上 | 按比例計算外推距離，最外邊不建柱 |
| 3 | 下構比上構大，最外邊在 Grid Line 上 | 往外一跨到下一 Grid Line，最外邊不建柱 |

### 5.3 判讀規則

- 同一案中，不同立面（東/西/南/北）可能分別對應不同情況，需逐面判斷
- 最外邊不建柱 = 該邊只有梁和板
- 外推距離必須從結構配置圖按比例計算，不可目測
- 無法判斷時，在「需確認事項」中列出

## 六、常見判讀情境

### 6.1 多種柱尺寸

同一樓層有不同尺寸的柱（邊柱較小、核心區較大），圖例中有多個柱項目，分別記錄。

### 6.2 梁尺寸沿跨度變化

同一 Grid Line 不同跨之間尺寸不同，需分段記錄：「Grid A / 1~3 = B55X80、Grid A / 3~5 = B60X90」

### 6.3 不規則平面（L型、U型）

某些 Grid 位置沒有柱或梁，如實記錄。缺角區域自然沒有梁圍合 → 不建板（建築物外）。

### 6.4 轉角柱 / L型柱

角落的異形柱在圖面有特殊標示，需注意讀取。

### 6.5 樓層間配置差異

不同樓層結構配置可能不同，每層獨立解讀，不從一層推斷另一層。

### 6.6 斜線寬度表示 (SB35/40X65)

- 表示兩種寬度，深度相同
- 可能用相同顏色，根據位置或標註區分
- 無法區分時統一記錄為 SB35/40X65

### 6.7 柱方向判斷

- 水平方向較長 = X向寬度較大
- 垂直方向較長 = Y向深度較大
- 正方形柱 X = Y

## Self-Learning Protocol

### 執行前：讀取經驗
載入本 skill 時，讀取 `learned/` 目錄中所有檔案作為補充知識。

### 執行後：紀錄新發現
任務完成後，檢查是否有以下新發現需要紀錄：

1. **patterns.md** — 新的命名模式、未見過的圖例符號、新的構件配置方式
2. **mistakes.md** — 本次犯的錯誤及修正方法（含根因分析）
3. **edge-cases.md** — 規則未覆蓋的特殊情況及處理決策

### 紀錄格式
每條紀錄包含：
- **日期**: YYYY-MM-DD
- **案名**: 專案識別
- **發現**: 具體描述
- **處理**: 採取的做法
- **是否應更新 SKILL.md**: Yes/No（如 Yes，標記待更新的 section）

### 紀錄原則
- 不重複紀錄已有的內容
- 先檢查 learned/ 現有內容再寫入
- 每個檔案保持 <100 行，超過時歸納合併舊條目
- 確認為通用規律後（>=2 次出現），才建議更新 SKILL.md 本體
