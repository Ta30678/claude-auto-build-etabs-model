# 結構構件辨識規則

本技能涵蓋結構配置圖中構件的辨識、圖例解讀、Grid 座標系統、特殊柱位處理。
斷面命名與解析規則已獨立至 `section-name` skill。

---

## 一、圖例讀取 (Legend)

### 1.1 Table-Based Legend

`pptx_to_elements.py` 自動搜尋 PPT 中的 2-column 表格解析圖例：

```
cell 0 = 背景色（構件顏色）
cell 1 = 斷面名稱文字
```

- **Phase filtering**: `--phase phase1` 排除 SB/FSB，`--phase phase2` 僅保留 SB/FSB
- **Theme Color (SCHEME) 解析**：PPT 使用 scheme color 時，工具自動解析為 RGB
- **色差容忍**：±15 RGB tolerance 匹配構件顏色

### 1.2 判讀要點

- **不假設顏色含義** — 紅色不一定是柱，藍色不一定是大梁，以表格為準
- 圖例中的文字 = 構件尺寸名稱，直接讀取
- 不同案子、不同樓層可能使用不同色彩

### 1.3 無表格時的處理

- **主動詢問使用者**確認對照關係

## 二、Grid 座標系統

### 2.1 基本規則

- Grid 以 ETABS `grid_data.json` 為準，PPT 僅驗證
- **不假設 X=數字、Y=字母**
- Grid Line 間距標示在圖面邊緣（單位 cm 或 mm）
- 圈號 (Grid Bubble) 為圓圈內的數字或字母

### 2.2 座標空間與資料流

構件座標經歷三個階段的轉換：

```
PPT-米座標        →  SLIDES INFO/{floor}/{floor}.json  (pptx_to_elements.py 提取)
Grid 錨點辨識     →  SLIDES INFO/{fl}/grid_anchors_{fl}.json  (READER 識別 Grid 標記)
Affine 校正       →  calibrated/{floor}/elements.json  (affine_calibrate.py --mode grid)
合併 + 驗證       →  elements.json                 (elements_merge.py + beam_validate.py)
```

- 初始提取階段的座標是 PPT-米座標，尚未對齊 Grid
- Grid 參照描述出現在 affine 校正之後的 `calibrated/` 和 `elements.json` 中

### 2.3 精度要求

座標精度要求 **1cm (0.01m)**。

## 三、構件辨識

### 3.1 構件類型速查表

| 圖面表現 | 構件類型 | 前綴 | 備註 |
|---------|---------|------|------|
| 填色方塊/矩形 | 柱 | C | 位於 Grid Line 交叉點 |
| 粗色線條 | 大梁 | B | 沿 Grid Line 連接柱與柱 |
| 色線條（較細） | 小梁 | SB | 位於大梁之間，不一定在 Grid Line 上 |
| 深色線（牆頂） | 壁梁 | WB | 位於剪力牆頂部 |
| 粗矩形色塊/色帶 | 剪力牆 | W | 沿大梁配置 |
| 外圍色帶 | 連續壁 | DW | 地下室外圍，材質 C280 |
| 大尺寸梁（基礎層） | 基礎梁 | FB | 僅基礎層 |

命名格式詳見 **section-name** skill。

### 3.2 牆的 PPT 2D 偏移

PPT 結構配置圖是 2D 展示：剪力牆/連續壁在圖面上畫在大梁**旁邊**（平行偏移），
因為牆和梁不能重疊在同一個 2D 位置。在 ETABS 3D 模型中，牆應建在梁**正下方**
（共用同一軸線座標）。

此偏移由 `beam_validate.py` 的 wall-to-beam snap 自動校正（tolerance=1.0m），
READER 在審閱 beam validation report 時確認 `wall_beam_snaps` 結果即可。

### 3.3 辨識流程

```
Step 1: 載入圖例對照表（自動提取或手動確認）
Step 2: 以顏色/色差匹配圖面上的構件
Step 3: 由位置判斷構件類型（Grid 交叉→柱、沿 Grid→梁、跨 Grid→牆）
Step 4: 記錄座標與尺寸
```

### 3.4 柱方向判斷

```
水平方向較長 = X向寬度較大
垂直方向較長 = Y向深度較大
正方形柱 X = Y
```

## 四、特殊處理

### 4.1 退縮柱位

柱子從某樓層向內退縮時：

- **必須詢問使用者**處理方式（轉換柱/取消柱/其他）
- 未獲指示前不自行決定

### 4.2 斜柱 / 斜撐 (BRACE)

- ETABS 中使用 **BRACE** 建立，不是 COLUMN
- 注意柱連續性不能斷開
- 起終點座標從圖面精確讀取
- 輸出中標註為 `BRACE`

### 4.3 柱連續性檢查

```
每根柱從基礎到屋頂應保持連續：
  B3F → B2F → B1F → 1F → 2F → ... → RF

如果某樓層柱位消失或偏移：
  1. 標記為「不連續」
  2. 詢問使用者處理方式
  3. 確認是否需要轉換梁/斜撐
```

### 4.4 共構下構範圍判讀

多棟共構（如 3 棟共用地下室）通常只建一棟模型。下構範圍比上構大，需判讀邊界處理方式。

**三種情況：**

| 情況 | 條件 | 處理 |
| ---- | ---- | ---- |
| 1 | 上構柱區已是最外邊 | 直接延續往下建 |
| 2 | 下構比上構大，最外邊不在 Grid Line 上 | 按比例計算外推距離，最外邊不建柱 |
| 3 | 下構比上構大，最外邊在 Grid Line 上 | 往外一跨到下一 Grid Line，最外邊不建柱 |

**判讀規則：**

- 同一案中，不同立面（東/西/南/北）可能分別對應不同情況，需逐面判斷
- 最外邊不建柱 = 該邊只有梁和板
- 外推距離必須從結構配置圖按比例計算，不可目測
- 無法判斷時，在「需確認事項」中列出

## 五、常見判讀情境

### 5.1 多種柱尺寸

同一樓層有不同尺寸的柱（邊柱較小、核心區較大），圖例中有多個柱項目，分別記錄。

### 5.2 梁尺寸沿跨度變化

同一 Grid Line 不同跨之間尺寸不同，需分段記錄：「Grid A / 1~3 = B55X80、Grid A / 3~5 = B60X90」

### 5.3 不規則平面（L型、U型）

某些 Grid 位置沒有柱或梁，如實記錄。缺角區域自然沒有梁圍合 → 不建板（建築物外）。

### 5.4 轉角柱 / L型柱

角落的異形柱在圖面有特殊標示，需注意讀取。

### 5.5 樓層間配置差異

不同樓層結構配置可能不同，每層獨立解讀，不從一層推斷另一層。

### 5.6 斜線寬度表示 (SB35/40X65)

- 表示兩種寬度，深度相同
- 可能用相同顏色，根據位置或標註區分
- 無法區分時統一記錄為 SB35/40X65

### 5.7 柱方向判斷

- 水平方向較長 = X向寬度較大
- 垂直方向較長 = Y向深度較大
- 正方形柱 X = Y

## Self-Learning Protocol

### 執行前：讀取經驗

載入本 skill 時，讀取 `learned/` 目錄中所有檔案作為補充知識。

### 執行後：紀錄新發現

任務完成後，檢查是否有以下新發現需要紀錄：

1. **patterns.md** — 新的圖例符號、新的構件配置方式
2. **mistakes.md** — 本次犯的錯誤及修正方法（含根因分析）
3. **edge-cases.md** — 規則未覆蓋的特殊情況及處理決策

### 紀錄格式

每條紀錄包含：

- **日期**: YYYY-MM-DD
- **案名**: 專案識別
- **發現**: 具體描述
- **處理**: 採取的做法
- **是否應更新 SKILL.md**: Yes/No（如 Yes，標記待更新的 section）

### 紀錄原則

- 不重複紀錄已有的內容
- 先檢查 learned/ 現有內容再寫入
- 每個檔案保持 <100 行，超過時歸納合併舊條目
- 確認為通用規律後（>=2 次出現），才建議更新 SKILL.md 本體
