# 結構配置圖通用解讀器

## 概述

本 Skill 用於**讀取並理解**結構工程師在建築平面圖上疊加繪製的結構配置圖。適用於所有專案，不綁定特定顏色或構件尺寸。

**核心原則：從圖面讀取資訊，不做假設。**

---

## 一、輸入來源與格式

結構配置圖通常以下列方式提供：

| 來源          | 格式            | 說明                          |
| ------------- | --------------- | ----------------------------- |
| Bluebeam Revu | PDF / 截圖 PNG  | 在建築平面 PDF 上疊加色彩標註 |
| PowerPoint    | PPTX / 截圖 PNG | 在平面圖上用 PPT 繪圖工具標註 |
| 手動截圖      | PNG / JPG       | 已完成標註的螢幕截圖          |

**一張完整的結構配置圖通常包含：**

1. 底圖：建築平面圖（含 Grid Line、尺寸標註、空間名稱）
2. 疊加標註：用不同顏色/線型表示各種結構構件
3. 圖例 (Legend)：說明每個顏色/符號代表的構件尺寸
4. Grid Line 圈號：標示在圖面邊緣（X方向通常用數字 1,2,3...，Y方向用英文字母 A,B,C...）

## 二、構件命名規則（通用於所有案子）

### 2.1 柱 (Column) — 前綴 "C"

```
格式：C{寬}X{深}
範例：C90X90, C100X110, C80X60, C50X50
```

- 「寬」= X方向長度 (cm)
- 「深」= Y方向長度 (cm)
- 圖面上通常以**填色方塊或矩形**表示
- 位於 Grid Line 交叉點上

### 2.2 大梁 / 主梁 (Main Beam) — 前綴 "B"

```
格式：B{寬}X{深}
範例：B55X80, B70X80, B60X90
```

- 「寬」= 梁寬 (cm)
- 「深」= 梁深/梁高 (cm)
- 圖面上通常以**粗色線條**表示
- 連接柱與柱之間（沿 Grid Line 方向）
- 大梁承載小梁和樓板的荷重

### 2.3 小梁 / 次梁 (Secondary Beam) — 前綴 "SB"

```
格式：SB{寬}X{深}  或  SB{寬1}/{寬2}X{深}
範例：SB35X65, SB30X60, SB25X50, SB35/40X65
```

- 當標示 SB35/40X65 時，表示同系列小梁有兩種寬度(35cm 和 40cm)，深度相同(65cm)
- 圖面上通常以**色線條**表示，線條可能比大梁細
- 位於大梁之間，不一定在 Grid Line 上
- **小梁的確切位置由結構工程師決定，Claude 需根據圖面實際位置判讀**

### 2.4 壁梁 (Wall Beam) — 前綴 "WB"

```
格式：WB{寬}X{深}
範例：WB50X70, WB40X60
```

- 壁梁通常位於剪力牆頂部
- 圖面上以**色線條**表示，有時顏色較深（棕色、深紅等）
- 與剪力牆位置重疊或相鄰

### 2.5 剪力牆 (Shear Wall)

```
格式：{厚度}cm 剪力牆  或  SW{厚度}
範例：50cm 剪力牆, 30cm 剪力牆, SW25
```

- 圖面上以**粗矩形色塊/色帶**表示
- 通常沿 Grid Line 配置
- 厚度以 cm 標示

### 2.6 基礎梁 (Foundation Beam) — 前綴 "FB"（僅基礎層）

```
格式：FB{寬}X{深}
範例：FB90X230, FB80X200
```

- 僅出現在基礎層（B1F 或 FL 平面圖）
- 尺寸通常較大（深度可達 200cm 以上）

### 2.7 基礎小梁 (Foundation Secondary Beam) — 前綴 "FSB"（僅基礎層）

```
格式：FSB{寬}X{深}
範例：FSB40X80
```

### 2.8 基礎邊梁 (Foundation Wall Beam) — 前綴 "FWB"（僅基礎層）

```
格式：FWB{寬}X{深}
範例：FWB40X80
```

## 三、圖例 (Legend) 解讀方法

### 3.1 解讀流程

```
步驟 1：在圖面上找到圖例區域
         → 通常位於圖面左側、右側、或下方空白處
         → 由一組「色塊/色線 + 文字」組成

步驟 2：逐項讀取圖例
         → 識別每個圖示的圖形類型（方塊 = 柱、線條 = 梁、粗帶 = 牆）
         → 讀取旁邊的文字標註（如 C90X90、B55X80）

步驟 3：建立「圖面顏色/圖示 → 構件」對照表
         → 此對照表僅適用於當前這張圖
         → 不同案子、不同樓層可能使用不同的色彩配置

步驟 4：用此對照表去辨識圖面上的所有構件
```

### 3.2 圖例判讀要點

- **方塊圖示** (■) → 柱 (Column)
- **線條圖示** (──) → 梁 (Beam)，依前綴區分 B/SB/WB/FB/FSB
- **粗帶/粗矩形** → 剪力牆 (Shear Wall)
- 圖例中的文字就是構件尺寸名稱，直接讀取即可
- **不要假設顏色含義** — 紅色不一定是柱，藍色不一定是梁，一切以圖例為準

### 3.3 無圖例時的處理

如果圖面上沒有明確的圖例：

1. 嘗試從圖面上的文字標註直接讀取構件名稱
2. 根據構件形狀推斷類型（方塊→柱、線條→梁）
3. **主動詢問用戶**確認構件對照關係

## 四、Grid Line 座標系統

### 4.1 基本規則

```
X方向 (水平)：通常用數字編號 → ①, ②, ③, ④, ⑤ ...
Y方向 (垂直)：通常用英文字母編號 → Ⓐ, Ⓑ, Ⓒ, Ⓓ ...
```

- Grid Line 間距標示在圖面邊緣（單位通常為 cm 或 mm）
- 圈號 (Grid Bubble) 為圓圈內的數字或字母

### 4.2 構件位置描述方式

```
柱位：   以兩條 Grid Line 的交叉點表示 → "Grid C-3 柱" 或 "(C,3)"
梁跨度： 以起終點的 Grid Line 表示 → "Grid A~D / Grid 2 大梁" 表示沿 Grid 2、從 A 到 D 的梁
牆位：   以 Grid Line 區段表示 → "Grid B~C / Grid 1 剪力牆"
```

### 4.3 讀取 Grid 間距

從圖面邊緣的尺寸標註讀取各 Grid Line 之間的距離：

- 例如：①→② = 860, ②→③ = 860, ③→④ = 860（單位 cm）
- 這些間距資訊對於判斷梁的跨度和構件配置非常重要

## 五、⚠️ 樓層對應規則（極度重要）

### 5.1 平面圖樓層 vs ETABS 樓層

```
┌─────────────────────────────────────────────────────────┐
│  ⚠️ 核心規則：柱的 ETABS 樓層 = 平面圖樓層 + 1        │
│                                                         │
│  平面圖 5F 的柱 → 對應 ETABS 6F 的柱                   │
│  平面圖 4F 的柱 → 對應 ETABS 5F 的柱                   │
│  平面圖 3F 的柱 → 對應 ETABS 4F 的柱                   │
│  ...以此類推                                            │
│                                                         │
│  原因：柱是「由下往上撐」的構件。                       │
│  平面圖標示的是「該樓層看到的柱截面」，                 │
│  但 ETABS 中柱歸屬於其「頂部」所在的樓層。              │
│  所以平面圖 5F 看到的柱，在 ETABS 中屬於 6F。           │
└─────────────────────────────────────────────────────────┘
```

### 5.2 各構件的樓層對應

| 構件類型      | 平面圖標示樓層 | ETABS 對應樓層 | 說明               |
| ------------- | -------------- | -------------- | ------------------ |
| **柱 (C)**    | NF             | **(N+1)F**     | 柱歸屬於其頂部樓層 |
| **大梁 (B)**  | NF             | **NF**         | 梁在哪層就是哪層   |
| **小梁 (SB)** | NF             | **NF**         | 同大梁             |
| **壁梁 (WB)** | NF             | **NF**         | 同大梁             |
| **剪力牆**    | NF             | **(N+1)F**     | 與柱相同邏輯       |

### 5.3 範例

```
用戶提供：「4F 結構配置圖」

讀取到的構件：
  柱 C90X90 在 Grid (B,2)
  → ETABS 中應查找：5F 的 (B,2) 柱

  大梁 B55X80 在 Grid A / 1~5
  → ETABS 中應查找：4F 的 Grid A 大梁

  剪力牆 50cm 在 Grid 1 / A~B
  → ETABS 中應查找：5F 的 Grid 1 / A~B 剪力牆
```

## 六、完整解讀流程

當用戶上傳結構配置圖時，依照以下步驟系統性解讀：

```
 Step 1  辨識圖面基本資訊
         ├─ 這是哪個樓層？（從標題或用戶說明得知）
         ├─ Grid Line 編號有哪些？（X向數字、Y向字母）
         └─ Grid Line 間距是多少？

 Step 2  讀取圖例 (Legend)
         ├─ 找到圖例位置
         ├─ 逐項記錄：「視覺特徵 → 構件名稱」
         │   例如：紅色方塊 → C90X90
         │         藍紫色粗線 → B55X80
         │         綠色線 → SB35/40X65
         └─ 建立本圖專用的對照表

 Step 3  辨識所有柱位
         ├─ 在圖面上找到所有柱的色塊
         ├─ 記錄每根柱的 Grid 位置 (如 A-1, B-3)
         ├─ 根據圖例對照表判斷柱尺寸
         └─ ⚠️ 記住：ETABS 樓層 = 平面圖樓層 + 1

 Step 4  辨識所有大梁
         ├─ 在圖面上找到所有大梁線條
         ├─ 判斷每根梁的方向（X向或Y向）
         ├─ 記錄梁的起終點 Grid Line
         ├─ 根據圖例對照表判斷梁尺寸
         └─ 大梁通常沿 Grid Line 配置

 Step 5  辨識所有小梁
         ├─ 在圖面上找到所有小梁線條
         ├─ 判斷方向和位置
         ├─ 小梁不一定在 Grid Line 上
         └─ 記錄相對位置（如「Grid 2~3 之間、沿Y向」）

 Step 6  辨識剪力牆和壁梁
         ├─ 找到粗帶狀色塊 → 剪力牆
         ├─ 找到與剪力牆重疊的線條 → 壁梁
         └─ ⚠️ 剪力牆 ETABS 樓層 = 平面圖樓層 + 1

 Step 7  輸出結構化摘要
         └─ 見下方「輸出格式」
```

## 七、輸出格式

解讀完成後，以下列格式整理結果：

```
## {樓層}F 結構配置摘要

### Grid 系統
- X方向：Grid ① ~ ⑤，間距 = 860, 860, 860, 860 (cm)
- Y方向：Grid Ⓐ ~ Ⓓ，間距 = 915, 2590, 915 (cm)

### 柱配置 (平面圖 {N}F → ETABS {N+1}F)
| Grid 位置 | 柱尺寸 |
|-----------|--------|
| A-1       | C90X90 |
| A-2       | C90X90 |
| ...       | ...    |

### 大梁配置 (ETABS {N}F)
| 方向 | Grid Line | 跨度 | 尺寸 |
|------|-----------|------|------|
| X    | Grid A    | 1~5  | B55X80 |
| Y    | Grid 2    | A~D  | B55X80 |
| ...  | ...       | ...  | ...    |

### 小梁配置 (ETABS {N}F)
| 方向 | 位置描述 | 尺寸 |
|------|---------|------|
| Y    | Grid 1~2 間 / 沿 Grid A~B | SB35/40X65 |
| ...  | ...     | ...  |

### 剪力牆配置 (平面圖 {N}F → ETABS {N+1}F)
| Grid 位置 | 厚度 |
|-----------|------|
| Grid 1 / A~B | 50cm |
| ...       | ...  |

### 壁梁配置 (ETABS {N}F)
| 位置 | 尺寸 |
|------|------|
| Grid 1 / A~B 上方 | WB50X70 |
| ...  | ...  |
```

## 八、常見判讀情境與處理

### 8.1 多種柱尺寸

同一樓層可能有不同尺寸的柱（如邊柱較小、核心區柱較大）。
→ 圖例中會有多個柱項目，分別記錄。

### 8.2 梁尺寸沿跨度變化

有時同一條 Grid Line 上的大梁在不同跨之間尺寸不同。
→ 需要分段記錄，如：「Grid A / 1~3 段 = B55X80、Grid A / 3~5 段 = B60X90」

### 8.3 不規則平面

有些建築平面不是矩形，可能有凸出或凹入。
→ 某些 Grid 位置可能沒有柱或梁，如實記錄即可。

### 8.4 轉角柱 / L型柱

某些角落的柱可能是 L 型或異形柱。
→ 通常在圖面上有特殊標示，需注意讀取。

### 8.5 樓層間配置差異

不同樓層的結構配置可能不同（如高樓層柱尺寸縮小）。
→ 每層獨立解讀，不要從一層推斷另一層。

### 8.6 圖例中的斜線表示 (SB35/40X65)

- `SB35/40X65` 表示有兩種寬度的小梁，深度相同
- 圖面上可能用相同顏色表示這兩種
- 可能需要根據位置或額外標註區分 35cm 和 40cm 的差異
- 若無法區分，統一記錄為 SB35/40X65

## 九、注意事項與禁忌

### ✅ 應該做的

- 從圖例讀取顏色與構件的對應關係
- 每張圖獨立建立對照表
- 明確區分平面圖樓層與 ETABS 樓層
- 記錄所有可見的構件，包含位置和尺寸
- 遇到無法辨識的項目時主動詢問用戶

### ❌ 不應該做的

- ❌ 假設「紅色一定是柱」或「藍色一定是大梁」
- ❌ 跨樓層推斷構件尺寸（每層獨立讀取）
- ❌ 忽略ETABS樓層規則（柱/牆的 ETABS 樓層 ≠ 平面圖樓層）
- ❌ 自行決定小梁位置（以圖面為準）
- ❌ 遺漏圖例中的任何構件類型

## 十、快速檢核清單

解讀完成後，用以下清單自我檢查：

```
□ 是否已讀取圖例中的所有構件類型？
□ 所有柱位是否都已記錄？
□ 所有大梁是否都已記錄（含方向和跨度）？
□ 小梁位置是否根據圖面實際位置記錄（非猜測）？
□ 剪力牆和壁梁是否已辨識？
□ 樓層對應是否正確？（柱/牆 → ETABS +1 層）
□ Grid 間距是否已記錄？
□ 是否有無法辨識的項目需要詢問用戶？
```
