# 結構配置圖通用解讀器

## 概述

本 Skill 用於**讀取並理解**結構工程師在建築平面圖上疊加繪製的結構配置圖。適用於所有專案，不綁定特定顏色或構件尺寸。

**核心原則：從圖面讀取資訊，不做假設。**

---

## 一、工作目錄與檔案管理

### 1.1 結構配置圖存放位置

結構配置圖的相關檔案（圖片、截圖、分析結果）統一存放在該 Case Folder 的 **「結構配置圖」** 子資料夾中：

```
{Case Folder}/
├── 結構配置圖/          <-- 所有結構配置圖相關檔案放這裡
│   ├── 1F.png           # 各樓層結構配置圖截圖
│   ├── 2F.png
│   ├── RF.png
│   └── ...
├── SPECTRUM.TXT         # 反應譜檔案（供 ETABS-Modeler 使用）
├── EQ_PARAMS.txt        # 地震力參數（供 ETABS-Modeler 使用）
└── ...
```

所有結構配置圖的分析工作都在此資料夾內完成。

---

## 二、輸入來源與格式

結構配置圖通常以下列方式提供：

| 來源          | 格式            | 說明                          |
| ------------- | --------------- | ----------------------------- |
| Bluebeam Revu | PDF / 截圖 PNG  | 在建築平面 PDF 上疊加色彩標註 |
| PowerPoint    | PPTX / 截圖 PNG | 在平面圖上用 PPT 繪圖工具標註 |
| 手動截圖      | PNG / JPG       | 已完成標註的螢幕截圖          |

**一張完整的結構配置圖通常包含：**

1. 底圖：建築平面圖（含 Grid Line、尺寸標註、空間名稱）
2. 疊加標註：用不同顏色/線型表示各種結構構件
3. 圖例 (Legend)：說明每個顏色/符號代表的構件尺寸
4. Grid Line 圈號：標示在圖面邊緣（X方向通常用數字 1,2,3...，Y方向用英文字母 A,B,C...）

## 三、構件命名規則（通用於所有案子）

### 3.1 柱 (Column) -- 前綴 "C"

```
格式：C{X向寬}X{Y向深}
範例：C90X90, C100X110, C80X60, C50X50
```

- **「X向寬」= Global X 方向（水平方向）的長度 (cm)**
- **「Y向深」= Global Y 方向（垂直方向）的長度 (cm)**
- 例如 `C120X180` 表示：水平方向 120cm，垂直方向 180cm
- 圖面上通常以**填色方塊或矩形**表示
- 位於 Grid Line 交叉點上

### 3.2 大梁 / 主梁 (Main Beam) -- 前綴 "B"

```
格式：B{寬}X{深}
範例：B55X80, B70X80, B60X90
```

- 「寬」= 梁寬 (cm)
- 「深」= 梁深/梁高 (cm)
- 圖面上通常以**粗色線條**表示
- 連接柱與柱之間（沿 Grid Line 方向）
- 大梁承載小梁和樓板的荷重

### 3.3 小梁 / 次梁 (Secondary Beam) -- 前綴 "SB"

```
格式：SB{寬}X{深}  或  SB{寬1}/{寬2}X{深}
範例：SB35X65, SB30X60, SB25X50, SB35/40X65
```

- 當標示 SB35/40X65 時，表示同系列小梁有兩種寬度(35cm 和 40cm)，深度相同(65cm)
- 圖面上通常以**色線條**表示，線條可能比大梁細
- 位於大梁之間，不一定在 Grid Line 上
- **小梁的確切位置必須從圖面計算得出**（見第六節「小梁位置計算方法」）

### 3.4 壁梁 (Wall Beam) -- 前綴 "WB"

```
格式：WB{寬}X{深}
範例：WB50X70, WB40X60
```

- 壁梁通常位於剪力牆頂部
- 圖面上以**色線條**表示，有時顏色較深（棕色、深紅等）
- 與剪力牆位置重疊或相鄰

### 3.5 剪力牆 (Shear Wall)

```
格式：{厚度}cm 剪力牆  或  SW{厚度}
範例：50cm 剪力牆, 30cm 剪力牆, SW25
```

- 圖面上以**粗矩形色塊/色帶**表示
- 通常沿 Grid Line 配置
- 厚度以 cm 標示

### 3.6 連續壁 (Diaphragm Wall / Retaining Wall)

```
格式：{厚度}cm 連續壁  或  DW{厚度}
範例：80cm 連續壁, 100cm 連續壁, DW80
```

- **材質一律使用 C280**（除非使用者另外指定）
- 通常出現在地下室外圍
- 圖面上以粗色帶表示

### 3.7 基礎梁 (Foundation Beam) -- 前綴 "FB"（僅基礎層）

```
格式：FB{寬}X{深}
範例：FB90X230, FB80X200
```

- 僅出現在基礎層（B1F 或 FL 平面圖）
- 尺寸通常較大（深度可達 200cm 以上）

### 3.8 基礎小梁 (Foundation Secondary Beam) -- 前綴 "FSB"（僅基礎層）

```
格式：FSB{寬}X{深}
範例：FSB40X80
```

### 3.9 基礎邊梁 (Foundation Wall Beam) -- 前綴 "FWB"（僅基礎層）

```
格式：FWB{寬}X{深}
範例：FWB40X80
```

## 四、圖例 (Legend) 解讀方法

### 4.1 解讀流程

```
步驟 1：在圖面上找到圖例區域
         -> 通常位於圖面左側、右側、或下方空白處
         -> 由一組「色塊/色線 + 文字」組成

步驟 2：逐項讀取圖例
         -> 識別每個圖示的圖形類型（方塊 = 柱、線條 = 梁、粗帶 = 牆）
         -> 讀取旁邊的文字標註（如 C90X90、B55X80）

步驟 3：建立「圖面顏色/圖示 -> 構件」對照表
         -> 此對照表僅適用於當前這張圖
         -> 不同案子、不同樓層可能使用不同的色彩配置

步驟 4：用此對照表去辨識圖面上的所有構件
```

### 4.2 圖例判讀要點

- **方塊圖示** -> 柱 (Column)
- **線條圖示** -> 梁 (Beam)，依前綴區分 B/SB/WB/FB/FSB
- **粗帶/粗矩形** -> 剪力牆 (Shear Wall) 或連續壁 (Diaphragm Wall)
- 圖例中的文字就是構件尺寸名稱，直接讀取即可
- **不要假設顏色含義** -- 紅色不一定是柱，藍色不一定是梁，一切以圖例為準

### 4.3 無圖例時的處理

如果圖面上沒有明確的圖例：

1. 嘗試從圖面上的文字標註直接讀取構件名稱
2. 根據構件形狀推斷類型（方塊->柱、線條->梁）
3. **主動詢問用戶**確認構件對照關係

## 五、Grid Line 座標系統

### 5.1 基本規則

```
X方向 (水平)：通常用數字編號 -> 1, 2, 3, 4, 5 ...
Y方向 (垂直)：通常用英文字母編號 -> A, B, C, D ...
```

- Grid Line 間距標示在圖面邊緣（單位通常為 cm 或 mm）
- 圈號 (Grid Bubble) 為圓圈內的數字或字母

### 5.2 構件位置描述方式

```
柱位：   以兩條 Grid Line 的交叉點表示 -> "Grid C-3 柱" 或 "(C,3)"
梁跨度： 以起終點的 Grid Line 表示 -> "Grid A~D / Grid 2 大梁" 表示沿 Grid 2、從 A 到 D 的梁
牆位：   以 Grid Line 區段表示 -> "Grid B~C / Grid 1 剪力牆"
```

### 5.3 讀取 Grid 間距（極重要 -- 小梁定位必須）

從圖面邊緣的尺寸標註讀取各 Grid Line 之間的距離：

- 例如：1->2 = 860, 2->3 = 860, 3->4 = 860（單位 cm）
- 計算各 Grid Line 的累積座標（用於小梁位置計算）

```
Grid 間距累積座標計算範例：
X方向 Grid: 1=0, 2=860, 3=1720, 4=2580 (cm)
Y方向 Grid: A=0, B=700, C=1400, D=2100 (cm)
```

**這些間距和座標是計算小梁位置的基礎，必須精確記錄。**

## 六、小梁位置計算方法（重大更新）

### 6.1 為什麼需要精確計算

小梁不在 Grid Line 上，位置由結構工程師根據樓板跨度和使用需求決定。
簡單的十字連接情況容易識別，但**交錯複雜的配置必須透過等比例計算才能正確建立**。

### 6.2 計算流程

```
Step 1: 建立 Grid 座標系
        -> 從圖面讀取所有 Grid Line 間距
        -> 計算各 Grid Line 的絕對座標（cm）

Step 2: 量測小梁在圖面上的像素位置
        -> 識別小梁線條的起點和終點
        -> 記錄像素座標 (px_x, px_y)

Step 3: 建立 Grid Line 的像素座標
        -> 找到圖面上至少 2 條已知 Grid Line 的像素位置
        -> 建立「像素 <-> 實際座標」的對應關係

Step 4: 等比例計算小梁的實際座標
        -> 利用線性內插，將像素位置轉換為實際座標

Step 5: 判斷小梁的起終點
        -> 確認小梁連接到哪些大梁或其他小梁
        -> 記錄精確的起終點座標
```

### 6.3 等比例計算公式

```
已知：
  Grid_A 的像素位置 = px_A，實際座標 = coord_A
  Grid_B 的像素位置 = px_B，實際座標 = coord_B
  小梁的像素位置 = px_SB

小梁的實際座標 = coord_A + (px_SB - px_A) / (px_B - px_A) * (coord_B - coord_A)
```

### 6.4 小梁位置輸出要求

每根小梁必須輸出：

| 欄位 | 說明 | 範例 |
|------|------|------|
| 方向 | X向或Y向 | Y |
| 起點Grid/座標 | 起點位置 | Grid A (Y=0) |
| 終點Grid/座標 | 終點位置 | Grid D (Y=2100cm) |
| 固定方向座標 | 垂直於梁方向的座標 | X=1290cm (Grid 2~3之間) |
| 所在區間 | 位於哪兩條Grid之間 | Grid 2~3 之間 |
| 尺寸 | 斷面尺寸 | SB35X65 |

### 6.5 常見小梁配置模式

```
模式 A：等間距配置
  -> 跨度內均勻分布 N 根小梁
  -> 間距 = 跨度 / (N+1)

模式 B：特定位置配置
  -> 小梁在特定比例位置（如 1/3、1/2、2/3）
  -> 需從圖面量測確認

模式 C：對齊上下層配置
  -> 小梁位置與其他樓層對齊
  -> 需交叉比對多層圖面

模式 D：交錯配置
  -> 相鄰跨的小梁交錯排列
  -> 每根小梁位置需獨立計算

模式 E：不規則配置
  -> 小梁位置因空間需求各不相同
  -> 每根必須從圖面個別計算
```

### 6.6 小梁連接判斷

```
小梁端點連接到：
  1. 大梁 -> 沿 Grid Line 的大梁
  2. 其他小梁 -> 在交叉處連接
  3. 剪力牆 -> 嵌入牆體
  4. 壁梁 -> 連接到壁梁

每個端點必須明確識別連接對象，
這決定了 ETABS 中小梁的起終點座標。
```

### 6.7 小梁連接性驗證（重要）

每根小梁的兩個端點都必須接觸到其他結構構件（大梁、小梁、柱、牆）。

**懸空小梁 = 可疑信號：**
- 如果小梁某端點沒有接觸任何構件 → 形成懸臂小梁
- 懸臂小梁極少見，通常只出現在陽台/露臺
- 發現懸空端點時，應先懷疑是否對圖面理解有誤
- 重新檢視該區域的圖面，確認小梁的起終點是否正確

**處理流程：**
1. 檢查每根小梁的兩個端點座標是否位於某條大梁/小梁/柱上
2. 若端點懸空 → 重新判讀該區域圖面
3. 若確認為陽台/露臺懸臂 → 在輸出中標記為「懸臂小梁」
4. 若無法確認 → 在「需確認事項」中列出

## 七、樓層對應規則（極度重要）

### 7.1 平面圖樓層 vs ETABS 樓層

```
核心規則：柱的 ETABS 樓層 = 平面圖樓層 + 1

  平面圖 5F 的柱 -> 對應 ETABS 6F 的柱
  平面圖 4F 的柱 -> 對應 ETABS 5F 的柱
  平面圖 3F 的柱 -> 對應 ETABS 4F 的柱
  ...以此類推

  原因：柱是「由下往上撐」的構件。
  平面圖標示的是「該樓層看到的柱截面」，
  但 ETABS 中柱歸屬於其「頂部」所在的樓層。
  所以平面圖 5F 看到的柱，在 ETABS 中屬於 6F。
```

### 7.2 各構件的樓層對應

| 構件類型      | 平面圖標示樓層 | ETABS 對應樓層 | 說明               |
| ------------- | -------------- | -------------- | ------------------ |
| **柱 (C)**    | NF             | **(N+1)F**     | 柱歸屬於其頂部樓層 |
| **大梁 (B)**  | NF             | **NF**         | 梁在哪層就是哪層   |
| **小梁 (SB)** | NF             | **NF**         | 同大梁             |
| **壁梁 (WB)** | NF             | **NF**         | 同大梁             |
| **剪力牆**    | NF             | **(N+1)F**     | 與柱相同邏輯       |
| **連續壁**    | NF             | **(N+1)F**     | 與柱相同邏輯       |

### 7.3 範例

```
用戶提供：「4F 結構配置圖」

讀取到的構件：
  柱 C90X90 在 Grid (B,2)
  -> ETABS 中應查找：5F 的 (B,2) 柱

  大梁 B55X80 在 Grid A / 1~5
  -> ETABS 中應查找：4F 的 Grid A 大梁

  剪力牆 50cm 在 Grid 1 / A~B
  -> ETABS 中應查找：5F 的 Grid 1 / A~B 剪力牆
```

## 七之一、樓層配置命名慣例

### 7a.1 樓層配置表示法

```
格式："{N}F/B{M}" 或 "{N}FB{M}"
  - {N}F = 上構到第 N 層（頂樓）
  - B{M} = 下構到 B{M}F，含基礎層
  - 頂樓以上還有屋突層：R1F, R2F, R3F, PRF（數量因案而異）

範例：
  "24F/B6" = 上構到 24F（頂樓），下構到 B6F（含基礎層）
  "15F/B3" = 上構到 15F（頂樓），下構到 B3F（含基礎層）
```

**每案樓層配置不同，禁止從記憶或過去案子推斷。必須從該 case 的 folder（樓層高度表或結構配置圖）取得實際樓層資訊。**

### 7a.2 屋突層定義（R1F ~ PRF）

```
屋突層 = 頂樓以上的樓層：R1F, R2F, R3F, PRF
數量因案而異，有些案子只有 R1F + PRF，有些有 R1F + R2F + R3F + PRF。

R1F 規則：
  - R1F 的樓板和梁配置 = 與頂樓相同
  - ETABS 柱從高樓層到低樓層往下長：
    - ETABS R1F 的柱 = 結構配置圖的「頂樓柱」
    - ETABS R2F 的柱 = 結構配置圖的「R1F 柱」
  - R1F 通常有結構配置圖可參考

R2F 以上規則：
  - 通常不會有 R2F 以上的結構配置圖
  - 做法：取 R1F 某些 Grid 區間（通常是電梯/樓梯核心區的 2x2 Grid 區間），直接延伸往上到 PRF
  - 延伸時使用相同的柱/梁/板配置
```

### 7a.3 讀圖時的注意事項

```
讀取結構配置圖時：
1. 頂樓的結構配置圖 → 同時用於 R1F 的梁/板配置
2. 頂樓的柱配置 → 對應 ETABS R1F 的柱（+1 規則）
3. R1F 如有專屬結構配置圖 → R1F 的柱配置對應 ETABS R2F 的柱
4. R2F 以上 → 詢問用戶哪些 Grid 區間需要延伸到 PRF
```

## 八、特殊柱位處理

### 8.1 退縮柱位

當柱子位置從某樓層開始向內退縮（常見於建築退縮設計）：

- **必須詢問使用者**要如何處理退縮柱位
- 可能的處理方式：
  1. 轉換柱（在退縮處用斜撐或轉換梁）
  2. 直接取消該柱（上方不再延伸）
  3. 其他使用者指定的方式
- **在未獲得使用者指示前，不要自行決定退縮柱位的處理方式**

### 8.2 斜柱 / 斜撐

- 斜柱在 ETABS 中使用 **"BRACE"** 建立，**不是** "COLUMN"
- 需注意柱子的**連續性**，不能斷開
- 斜撐的起終點座標需從圖面精確讀取
- 在輸出中標註為 `BRACE`，並記錄傾斜方向和起終點

### 8.3 柱連續性檢查

```
每根柱從基礎到屋頂應保持連續：
  B3F -> B2F -> B1F -> 1F -> 2F -> ... -> RF

如果某樓層柱位消失或偏移：
  1. 標記為「不連續」
  2. 詢問使用者處理方式
  3. 確認是否需要轉換梁/斜撐
```

## 八之一、共構下構範圍判讀

### 8a.1 背景

多棟共構（如 3 棟共用地下室）通常只建一棟模型。下構（地下室）的範圍通常比上構大，需要判讀邊界如何處理。

### 8a.2 三種情況

```
情況 1：高樓柱區已是最外邊
  - 上構延伸下來的柱已在結構物最外圍
  - 直接延續往下建，不需額外處理

情況 2：下構比上構大，且最外邊不在 Grid Line 上
  - 將下構延伸到最外邊
  - 最外邊不建柱
  - 外推距離：從結構配置圖上其他 Grid Line 間距，按比例計算
  - 例如：若從圖面量測，最外邊距最近 Grid Line 約 0.4 跨距，則按比例計算實際距離

情況 3：下構比上構大，且最外邊在 Grid Line 上
  - 將下構往外建到下一跨（下一個 Grid Line）
  - 最外邊不建柱
  - 此情況包含不同棟下構的連接處
```

### 8a.3 判讀規則

```
1. 同一案中，結構物的不同立面（東/西/南/北）可能分別對應不同情況，需逐面判斷
2. 最外邊不建柱 = 該邊只有梁和板，柱到上構範圍為止
3. 外推距離必須從結構配置圖按比例計算，不可用目測估計
4. 若無法判斷屬於哪種情況，在「需確認事項」中列出，詢問使用者
```

## 九、完整解讀流程

當用戶上傳結構配置圖時，依照以下步驟系統性解讀：

```
 Step 1  辨識圖面基本資訊
         |-- 這是哪個樓層？（從標題或用戶說明得知）
         |-- Grid Line 編號有哪些？（X向數字、Y向字母）
         |-- Grid Line 間距是多少？
         +-- 計算各 Grid Line 的累積座標

 Step 2  讀取圖例 (Legend)
         |-- 找到圖例位置
         |-- 逐項記錄：「視覺特徵 -> 構件名稱」
         |   例如：紅色方塊 -> C90X90
         |         藍紫色粗線 -> B55X80
         |         綠色線 -> SB35/40X65
         +-- 建立本圖專用的對照表

 Step 3  辨識所有柱位
         |-- 在圖面上找到所有柱的色塊
         |-- 記錄每根柱的 Grid 位置 (如 A-1, B-3)
         |-- 根據圖例對照表判斷柱尺寸
         |-- C{X向寬}X{Y向深}：X向=水平，Y向=垂直
         |-- 檢查是否有退縮柱位或斜柱 -> 若有，詢問使用者
         +-- 記住：ETABS 樓層 = 平面圖樓層 + 1

 Step 4  辨識所有大梁
         |-- 在圖面上找到所有大梁線條
         |-- 判斷每根梁的方向（X向或Y向）
         |-- 記錄梁的起終點 Grid Line
         |-- 根據圖例對照表判斷梁尺寸
         +-- 大梁通常沿 Grid Line 配置

 Step 5  辨識並計算所有小梁位置（精確定位）
         |-- 在圖面上找到所有小梁線條
         |-- 判斷方向（X向或Y向）
         |-- 利用 Grid Line 間距進行等比例計算（見第六節）
         |-- 記錄每根小梁的精確座標（cm）
         |-- 確認起終點連接到哪些構件
         +-- 不確定的位置用「約」標記，並在輸出中提醒

 Step 6  辨識剪力牆、連續壁和壁梁
         |-- 找到粗帶狀色塊 -> 剪力牆或連續壁
         |-- 連續壁材質 = C280（預設）
         |-- 找到與剪力牆重疊的線條 -> 壁梁
         +-- 剪力牆/連續壁 ETABS 樓層 = 平面圖樓層 + 1

 Step 7  輸出結構化摘要（見第十節「輸出格式」）
```

## 十一、常見判讀情境與處理

### 11.1 多種柱尺寸

同一樓層可能有不同尺寸的柱（如邊柱較小、核心區柱較大）。
-> 圖例中會有多個柱項目，分別記錄。

### 11.2 梁尺寸沿跨度變化

有時同一條 Grid Line 上的大梁在不同跨之間尺寸不同。
-> 需要分段記錄，如：「Grid A / 1~3 段 = B55X80、Grid A / 3~5 段 = B60X90」

### 11.3 不規則平面

有些建築平面不是矩形，可能有凸出或凹入。
-> 某些 Grid 位置可能沒有柱或梁，如實記錄即可。

### 11.4 轉角柱 / L型柱

某些角落的柱可能是 L 型或異形柱。
-> 通常在圖面上有特殊標示，需注意讀取。

### 11.5 樓層間配置差異

不同樓層的結構配置可能不同（如高樓層柱尺寸縮小）。
-> 每層獨立解讀，不要從一層推斷另一層。

### 11.6 圖例中的斜線表示 (SB35/40X65)

- `SB35/40X65` 表示有兩種寬度的小梁，深度相同
- 圖面上可能用相同顏色表示這兩種
- 可能需要根據位置或額外標註區分 35cm 和 40cm 的差異
- 若無法區分，統一記錄為 SB35/40X65

### 11.7 柱方向判斷

- 從圖面上柱的矩形形狀判斷 X/Y 向尺寸
- 水平方向較長 = X向寬度較大
- 垂直方向較長 = Y向深度較大
- 正方形柱 X=Y

## 十二、注意事項與禁忌

### 鐵則（ABSOLUTE RULE — 違反即失敗）

**小梁位置絕對禁止用 1/2、1/3 grid 間距猜測或假設等間距配置！**
必須逐根從圖面量測像素位置，用 Grid Line 像素座標做等比例插值，計算出精確座標（cm 級精度）。
小梁位置由結構工程師根據住宅單元隔間決定，每根位置都不同。
用等分假設建模 = 工程師失職，等同於造假數據。
如果圖面解析度不足以量測，必須回報「無法量測」並請求更清晰的圖面，絕不可用假設值代替。

### 應該做的

- 從圖例讀取顏色與構件的對應關係
- 每張圖獨立建立對照表
- 明確區分平面圖樓層與 ETABS 樓層
- 記錄所有可見的構件，包含位置和尺寸
- **利用 Grid 間距等比例計算小梁位置（精確到 cm）**
- **記錄柱尺寸時注意 X/Y 方向對應**
- **連續壁預設材質 C280**
- **遇到退縮柱位時主動詢問使用者**
- **斜柱標記為 BRACE**
- 遇到無法辨識的項目時主動詢問用戶
- 輸出座標資訊供 ETABS-Modeler 使用

### 不應該做的

- 假設「紅色一定是柱」或「藍色一定是大梁」
- 跨樓層推斷構件尺寸（每層獨立讀取）
- 忽略ETABS樓層規則（柱/牆的 ETABS 樓層 ≠ 平面圖樓層）
- **僅描述小梁的大概位置（如「Grid 2~3 之間」）而不計算精確座標**
- **用 1/2、1/3 等分假設來放置小梁（這是鐵則禁令，違反即失敗）**
- **從舊模型複製或按比例縮放小梁位置到新模型**
- **將退縮柱位自行處理而不詢問使用者**
- **將斜柱當作一般柱處理（應為 BRACE）**
- 遺漏圖例中的任何構件類型
- 混淆柱的 X/Y 向尺寸
