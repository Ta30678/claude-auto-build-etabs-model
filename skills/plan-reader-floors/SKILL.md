# 結構樓層對應規則與樓板判斷

本技能涵蓋樓層對應規則、屋突層定義、樓板建板判斷、小梁位置備案量測。

---

## 一、樓層對應規則與命名

### 1.1 平面圖樓層 vs ETABS 樓層

```
核心規則：柱/牆的 ETABS 樓層 = 平面圖樓層 + 1

  平面圖 5F 的柱 → ETABS 6F 的柱
  平面圖 4F 的柱 → ETABS 5F 的柱

  原因：柱「由下往上撐」。平面圖標示的是該樓層看到的柱截面，
  ETABS 中柱歸屬於頂部所在樓層。
```

### 1.2 各構件的樓層對應

| 構件類型 | 平面圖標示 | ETABS 對應 | 說明 |
|----------|-----------|-----------|------|
| 柱 (C) | NF | **(N+1)F** | 柱歸屬於頂部樓層 |
| 大梁 (B) | NF | **NF** | 梁在哪層就是哪層 |
| 小梁 (SB) | NF | **NF** | 同大梁 |
| 壁梁 (WB) | NF | **NF** | 同大梁 |
| 剪力牆 | NF | **(N+1)F** | 與柱相同邏輯 |
| 連續壁 | NF | **(N+1)F** | 與柱相同邏輯 |

### 1.3 範例

```
用戶提供：「4F 結構配置圖」

柱 C90X90 在 Grid (B,2) → ETABS 5F 的 (B,2) 柱
大梁 B55X80 在 Grid A / 1~5 → ETABS 4F 的 Grid A 大梁
剪力牆 50cm 在 Grid 1 / A~B → ETABS 5F 的 Grid 1 / A~B 剪力牆
```

### 1.4 樓層配置命名慣例

```
格式："{N}F/B{M}"
  {N}F = 上構到第 N 層（頂樓）
  B{M} = 下構到 B{M}F，含基礎層
  頂樓以上有屋突層：R1F, R2F, R3F, PRF

範例：
  "24F/B6" = 上構到 24F，下構到 B6F
  "15F/B3" = 上構到 15F，下構到 B3F
```

**每案樓層配置不同，禁止從記憶或過去案子推斷。必須從該 case folder 取得實際樓層資訊。**

### 1.5 屋突層定義（R1F ~ PRF）

```
屋突層 = 頂樓以上的樓層：R1F, R2F, R3F, PRF
數量因案而異。

預設屋突 = R1F~R3F + PRF，每層 3m，PRF 高度 = 最後屋突層高度（3m）。
  如 PPT 沒有屋突頁面，自動加入預設屋突。

R1F 規則：
  - R1F 的樓板和梁配置 = 與頂樓完全相同（所有梁/版/小梁，不分核心/非核心）
  - ETABS R1F 的柱 = 結構配置圖的「頂樓柱」（+1 規則）
  - ETABS R2F 的柱 = 結構配置圖的「R1F 柱」

R2F 以上規則：
  - 通常沒有 R2F 以上的結構配置圖
  - R2F~PRF = 只有核心區（電梯/樓梯區域的構件）
  - 做法：取 R1F 某些 Grid 區間（電梯/樓梯核心區，約 2x2 Grid），延伸到 PRF
  - 延伸時使用相同柱/梁/板配置
```

### 1.7 floors 欄位與 +1 Rule 的關係

model_config.json 的 `floors` 陣列語意因構件類型而異：

| 構件 | floors 語意 | Golden Scripts 行為 | 記憶口訣 |
|------|------------|-------------------|---------|
| **柱/牆** | 構件「站立的樓層」 | floor N → 建構件從 N 到 next_story(N) | 從這層「站起來」 |
| **梁/版/小梁** | 構件「坐落的樓層」 | floor N → 建構件在 N 標高 | 「坐在」這層 |

**常見錯誤**：AI 誤將 +1 後的頂端層寫進柱/牆的 floors，導致多建一層。
- 柱在 B3F~14F 圖面 → floors 最後寫 `14F`（+1 自動到 R1F），**不可**寫 `R1F`
- 連續壁在 B3F~B1F → floors 最後寫 `B1F`（+1 自動到 1F），**不可**寫 `1F`

### 1.6 讀圖時的注意事項

```
1. 頂樓結構配置圖 → 同時用於 R1F 的梁/板配置
2. 頂樓柱配置 → 對應 ETABS R1F 的柱（+1 規則）
3. R1F 如有專屬結構配置圖 → R1F 柱配置對應 ETABS R2F 的柱
4. R2F 以上 → 詢問用戶哪些 Grid 區間延伸到 PRF
```

## 二、SB 位置計算（備案流程）

> **此流程僅在無 annotation.json 時使用（如 PPT 來源案件）。annotation.json 為主要來源。**

### 2.1 為什麼需要精確計算

小梁不在 Grid Line 上，位置由結構工程師根據樓板跨度和使用需求決定。交錯複雜的配置必須透過等比例計算才能正確建立。

### 2.2 計算流程

```
Step 1: 建立 Grid 座標系
        → 從圖面讀取所有 Grid Line 間距
        → 計算各 Grid Line 絕對座標（cm）

Step 2: 量測小梁在圖面上的像素位置
        → 識別小梁線條起點和終點
        → 記錄像素座標 (px_x, px_y)

Step 3: 建立 Grid Line 的像素座標
        → 至少 2 條已知 Grid Line 的像素位置
        → 建立「像素 ↔ 實際座標」對應

Step 4: 等比例計算小梁實際座標
        → 線性內插：像素位置 → 實際座標

Step 5: 判斷小梁起終點
        → 確認連接到哪些大梁或其他小梁
        → 記錄精確起終點座標
```

### 2.3 等比例計算公式

```
已知：
  Grid_A 像素位置 = px_A，實際座標 = coord_A
  Grid_B 像素位置 = px_B，實際座標 = coord_B
  小梁像素位置 = px_SB

小梁實際座標 = coord_A + (px_SB - px_A) / (px_B - px_A) * (coord_B - coord_A)
```

### 2.4 小梁位置輸出要求

| 欄位 | 說明 | 範例 |
|------|------|------|
| 方向 | X向或Y向 | Y |
| 起點Grid/座標 | 起點位置 | Grid A (Y=0) |
| 終點Grid/座標 | 終點位置 | Grid D (Y=2100cm) |
| 固定方向座標 | 垂直於梁方向的座標 | X=1290cm |
| 所在區間 | 位於哪兩條Grid之間 | Grid 2~3 之間 |
| 尺寸 | 斷面尺寸 | SB35X65 |

### 2.5 常見小梁配置模式

| 模式 | 描述 |
|------|------|
| A 等間距 | 跨度內均勻分布 N 根，間距 = 跨度/(N+1) |
| B 特定位置 | 1/3、1/2、2/3 位置，需從圖面量測確認 |
| C 對齊上下層 | 與其他樓層對齊，需交叉比對多層圖面 |
| D 交錯配置 | 相鄰跨交錯排列，每根獨立計算 |
| E 不規則 | 因空間需求各不同，每根從圖面個別計算 |

### 2.6 小梁連接性驗證

每根小梁的兩個端點都必須接觸其他結構構件（大梁、小梁、柱、牆）。

**懸空小梁 = 可疑信號**（懸臂僅出現在陽台/露臺）。

處理流程：
1. 檢查每根小梁兩端是否位於某條大梁/小梁/柱上
2. 若端點懸空 → 重新判讀該區域圖面
3. 若確認為懸臂 → 標記為「懸臂小梁」
4. 若無法確認 → 在「需確認事項」中列出

## 三、樓板區域判斷

### 3.1 打叉叉（X）標記辨識

圖面上部分區域標示打叉叉（X），表示開孔、管道間、梯間、電梯井等：
- 兩條對角線交叉形成 X
- 可能附帶文字（「開孔」「管道間」「EL」「ST」）

### 3.2 決策矩陣

|              | 四面梁圍合 | 無四面梁 |
|-------------|----------|---------|
| **無打叉**   | 建板     | 不建板（建築物外） |
| **有打叉**   | 不建板（結構開孔） | 建板（管道/排氣標記） |

**邏輯**：
- 四面梁 + 無打叉 = 正常樓板 → 建板
- 四面梁 + 有打叉 = 結構開孔（梯間/電梯井）→ 不建板
- 無四面梁 + 無打叉 = 建築物外部 → 不建板
- 無四面梁 + 有打叉 = 管道標記，樓板仍在 → 建板

### 3.3 判斷流程

```
Step 1: 掃描圖面，找出所有打叉叉區域
        → 記錄所在 Grid 範圍（如 A-B x 2-3）

Step 2: 對每個 Grid 區域，檢查是否有四面梁圍合
        → 上下左右各有一條梁（大梁、小梁、壁梁均算）

Step 3: 套用決策矩陣，標記建板/不建板

Step 4: 輸出「樓板區域判斷」表格
```

### 3.4 不規則建築形狀

L型、U型等非矩形建築，在缺角處自然沒有梁圍合。決策矩陣已自動涵蓋：缺角處無梁 + 無打叉 → 不建板。

## 四、常見樓層情境

### 4.1 基礎層配置

基礎層（B1F 或 FL 平面圖）出現 FB（基礎梁）、FS（基礎版）、FSB（基礎小梁）、FWB（基礎邊梁），尺寸通常較大。

### 4.2 1F 室內外分區

1F 範圍可能分為室內（上構範圍內）和室外（上構範圍外），載重不同。需從建築平面圖判斷分界。

### 4.3 頂樓與屋突層的關係

頂樓配置圖同時用於 R1F 的梁/板配置。注意 R1F 柱 = 頂樓圖面柱（+1 規則），R2F 以上通常無圖，用核心區延伸。

### 4.4 地下室外擴層

下構範圍比上構大時，外擴區域的構件配置需特別注意。最外邊不建柱，梁和板需延伸。

## Self-Learning Protocol

### 執行前：讀取經驗
載入本 skill 時，讀取 `learned/` 目錄中所有檔案作為補充知識。

### 執行後：紀錄新發現
任務完成後，檢查是否有以下新發現需要紀錄：

1. **patterns.md** — 新的命名模式、未見過的圖例符號、新的構件配置方式
2. **mistakes.md** — 本次犯的錯誤及修正方法（含根因分析）
3. **edge-cases.md** — 規則未覆蓋的特殊情況及處理決策

### 紀錄格式
每條紀錄包含：
- **日期**: YYYY-MM-DD
- **案名**: 專案識別
- **發現**: 具體描述
- **處理**: 採取的做法
- **是否應更新 SKILL.md**: Yes/No（如 Yes，標記待更新的 section）

### 紀錄原則
- 不重複紀錄已有的內容
- 先檢查 learned/ 現有內容再寫入
- 每個檔案保持 <100 行，超過時歸納合併舊條目
- 確認為通用規律後（>=2 次出現），才建議更新 SKILL.md 本體
