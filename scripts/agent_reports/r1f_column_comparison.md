## CRITICAL BUGS FOUND

### Bug 1: Section Dimensions B Not Converted to Meters

In the MERGED e2k file (which uses TON/M units), several column sections have
the D dimension correctly converted to meters but the B dimension is still in cm:

| Section | D (MERGED) | B (MERGED) | Expected D (m) | Expected B (m) |
|---------|------------|------------|----------------|----------------|
| C100X100C42 | 1.0 | 100.0 | 1.0 | 1.0 |
| C100X100C420 | 1.0 | 100.0 | 1.0 | 1.0 |
| C100X100C420SD490 | 1.0 | 100.0 | 1.0 | 1.0 |
| C130X130C42 | 1.3 | 130.0 | 1.3 | 1.3 |
| C130X130C420 | 1.3 | 130.0 | 1.3 | 1.3 |
| C130X130C420SD490 | 1.3 | 130.0 | 1.3 | 1.3 |
| C150X150C42 | 1.5 | 150.0 | 1.5 | 1.5 |
| C150X150C420 | 1.5 | 150.0 | 1.5 | 1.5 |
| C150X150C420SD490 | 1.5 | 150.0 | 1.5 | 1.5 |

**Total affected sections: 9**

These sections all come from buildings A, C, D (which use KGF/CM units).
When merging into the MERGED e2k (TON/M), the D dimension was converted by dividing
by 100, but B was left unconverted. This means ETABS would interpret these columns
as extremely thin and wide rectangles instead of squares.

## Building B Special Notes

Building B uses SRC (Steel Reinforced Concrete) columns with Auto Select Lists.
At stories 30F-34F, it uses `SRC100X100C420` (D=1.0m, B=1.0m = 100x100cm).
At R1F and above, it transitions to SRC auto-select sections:

| Section | Type | Size Range (mm) |
|---------|------|-----------------|
| ACB3 | Auto Select | 500x500 to 1000x1200 |
| ACB4 | Auto Select | 500x500 to 1000x1000 |
| ACBC2 | Auto Select | 500x1100 to 3000x4000 |
| ACN1 | Auto Select | 500x500 to 1300x1800 |
| SRC100X100C420 | Rectangular | 1000x1000 (100x100cm) |

In the MERGED model, B building columns at R1F correctly use the same
auto-select section names (ACB3, ACB4, ACBC2, ACN1) as the original.
These sections are preserved correctly in the merge.
